# 写真台帳

設計事務所の工事写真台帳を作るための、ブラウザだけで完結するWebアプリです。サーバーは不要で、社内サーバーでも Vercel のような静的ホスティングでも、ローカルで `npm run dev` して開くだけでも使えます。

## できること

- **ドラッグ&ドロップで取り込み**：写真（フォルダごとも可）をドロップするだけで台帳に追加されます。
- **iPhoneのHEIC/HEIF対応**：拡張子の違いで表示できない `.heic` / `.heif` は、取り込み時に自動でJPEGへ変換します。
- **写真サイズは固定表示**：台帳・PDFとも、写真は縦横比を保ったまま中央トリミングで統一サイズに揃えます。
- **自動改ページ**：物件ごとにグループ化し、1ページ6枚を超えると自動で次のページに送ります。
- **撮影日の自動取得**：EXIFの撮影日時を読み取り、無ければファイルの更新日時を使います。
- **物件名の自動入力**：フォルダごとドロップすると、フォルダ名がそのまま物件名になります（既存の物件名と一致すれば統合されます）。
- **説明文の初期値を自動生成**：ファイル名から簡易的に説明文の初期値を作ります（例: `01_外壁_南面.jpg` → `外壁 南面`）。手動で自由に編集できます。
- **元写真をドライブに保存**：Googleドライブ デスクトップアプリの同期フォルダをローカルフォルダとして選択すると、物件名のサブフォルダに元データをそのまま保存します（OAuth設定不要）。File System Access APIに対応していないブラウザでは、代わりにZIPダウンロードで同じ構成を作れます。
- **検索**：物件名・説明・撮影日・ファイル名でその場に絞り込めます。
- **PDF提出**：物件ごとに1つの台帳PDFとして出力できます。

## データの保存場所

台帳データ（写真・説明・物件名など）はブラウザのIndexedDBにローカル保存されます。特定のサーバーやアカウントに依存しないので、「管理はどこでも良い」という要件どおり、このアプリをどこでホストしても、開いたブラウザの中にデータが残ります。別の端末に移したい場合は、PDFやZIPでの書き出しをご利用ください。

## セットアップ

```bash
npm install
npm run dev      # 開発サーバー起動
npm run build    # 本番ビルド（dist/ に出力、そのまま静的ホスティング可能）
npm run lint      # oxlint
```

## 技術構成

- Vite + React + TypeScript（クライアント完結、バックエンドなし）
- `heic2any`：HEIC/HEIF → JPEG変換
- `exifr`：EXIFからの撮影日抽出
- `idb`：IndexedDBラッパー（写真・物件・設定の保存）
- `jspdf`：台帳PDF生成
- `jszip`：Drive同期フォルダが使えないブラウザ向けのZIP書き出し
- File System Access API：Googleドライブなどの同期フォルダへの直接書き込み（対応ブラウザ: Chrome / Edge など）

## ブラウザ対応について

「ドライブ同期フォルダを選択」機能は File System Access API を使用しており、Chrome / Edge 系ブラウザで動作します。Firefox / Safari など未対応のブラウザでは、代わりに「元写真をZIPで保存」ボタンが表示され、ダウンロードしたZIPを手動でGoogleドライブの同期フォルダに入れる運用になります。
